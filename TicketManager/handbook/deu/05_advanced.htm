5.1 Regelmäßige Erinnerungs-Emails versenden

Wenn in einem Projekt mehr und mehr Fehler entdeckt werden, können Entwickler in diesem Projekt möglicherweise Bugs die ihnen zugewiesen wurden vergessen. Wenn sie den Bug Tracker nicht ständig besuchen und auf ihnen zugewiesene Fehler prüfen, können sie diese möglicherweise auch einfach übersehen und so die Terminplanung des Projekts verzögern.

Um das zu vermeiden, können Sie den Bug Tracker so konfigurieren, dass er regelmäßig Erinnerungsmails verschickt. Ín der Standardeinstellung nimmt Bugtracker alle offenen Fehlerberichte die als Priorität "sehr hoch" haben und schickt eine Mail an denjenigen in "Zugewiesen an".

Wenn Sie Bugtracker auf einer UNIX-Plattform betreiben, können Sie die Datei "/etc/crontab" bearbeiten und die folgende Zeile hinzufügen um täglich eine Erinnerungsmail zu verschicken.

5   4   *   *   *   root  /home/www/bug/crontab/mail_high_priority.php

Das obige Kommando führt die Datei "/home/www/bug/crontab/mail_high_priority.php"  jede Nacht aus. Sie müssen den Pfad entsprechend an den Pfad ihres Bugtracker-Verzeichnis anpassen. Dieser Befehl erfordert das PHP CLI (Kommandozeilenumgebung)  (So kann PHP von der Kommandozeile angesprochen werden, statt von der Webseite. Wenn das PHP CLI auf ihrem System nicht verfügbar ist, können Sie auch versuchen dieses Kommando in "lynx -dump http://127.0.0.1/bug/crontab/mail_high_priority.php" zu ändern, dadurch wird versucht lynx zu verwenden um sich auf die Seite "mail_high_priority.php" zu verbinden.

Bitte beachten Sie, das Sie das PHP CLI Kommando oder einen lokalen Browser verwenden sollten, um auf "mail_high_priority.php" zuzugreifen. Fernzugriff ist nicht erlaubt.

5.2 Anpassung an andere Sprachen

Momentan (Version 2.1.4) unterstützt der Bug Tracker nur Englisch, Traditionelles Chinesisch, Vereinfachtes Chinesisch, Polnisch, Deutsch und Italienisch. Sie können aber ihre eigene Sprache hinzufügen. Um eine Anpassung an ihre Sprache durchzuführen, kopieren Sie  bitte die Englische String Datei und ändern Sie entsprechend. Bitte beachten Sie, das die String-Datei im UTF-8 Format vorliegt. Sie müssen Sie mit einem Editor bearbeiten, der UTF-8 unterstützt. (Beispielsweise EmEditor, Microsoft Word oder UltraEdit 10.) Bitte denken Sie daran die Datei anschließend auch im UTF-8-Format zu speichern.

Für die folgenden Sprachen ist die Unterstützung in Bug Tracker bereits vorbereitet:

Sprache Abkürzung Stringdatei
Arabisch ar setup/strings/string.ar
Traditionelles Chinesisch cht setup/strings/string.cht
Vereinfachtes Chinesisch chs setup/strings/string.chs
Tschechisch cs setup/strings/string.cs
Dänisch da setup/strings/string.da
Deutsch de setup/strings/string.de
Englisch en setup/strings/string.en
Spanisch es setup/strings/string.es
Finnisch fi setup/strings/string.fi
Französisch fr setup/strings/string.fr
Hebräisch he setup/strings/string.he
Isländisch is setup/strings/string.is
Italienisch it setup/strings/string.it
Japanisch ja setup/strings/string.ja
Koreanisch ko setup/strings/string.ko
Niederländisch nl setup/strings/string.nl
Norwegisch no setup/strings/string.no
Polnisch pl setup/strings/string.pl
Portugiesisch pt setup/strings/string.pt
Russisch ru setup/strings/string.ru
Schwedisch sv setup/strings/string.sv
Slowakisch sk setup/strings/string.sk
Thailändisch th setup/strings/string.th
Ukrainisch uk setup/strings/string.uk

Wenn ihre Sprache in der Tabelle oben enthalten ist, müssen Sie nur die Englische Stringdatei kopieren, in ihre Sprache übersetzen und entsprechend in ihr Verzeichnis kopieren.

Wenn ihre Sprache nicht in der Tabelle enthalten ist, müssen sie die Datei "setup/string.php" ändern. Suchen Sie nach "$lang_array" und fügen Sie ihre Sprache hinzu.

/* To add your own string file: */
$lang_array = array("ar" => "Arabic",
	"cht" => "Chinese Traditional", 
	"chs" => "Chinese Simplified", 
	"cs" => "Czech",
	"da" => "Danish",
	"de" => "German",
	"en" => "English", 
	"es" => "Spanish", 
	"fi" => "Finnish",
	"fr" => "French", 
	"he" => "Hebrew",
	"is" => "Icelandic",
	"it" => "Italian",
	"ja" => "Japanese", 
	"ko" => "Korean",
	"nl" => "Dutch",
	"no" => "Norwegian",
	"pl" => "Polish", 
	"pt" => "Portuguese",
	"ru" => "Russian", 
	"sv" => "Swedish",
	"sk" => "Slovak",
	"th" => "Thai",
	"uk" => "Ukrainian"
);

Als Beispiel: Sie wollen eine Stringdatei für Japanisch hinzufügen. Kopieren Sie einfach setup/string/string.en nach setup/string/string.ja und übersetzen Sie dieses nach Japanisch

Nachdem Sie die Übersetzung beendet haben, verbinden Sie sich mit ihrem Browser auf die URL http://your.domain.com/bug/setup/string.php. (Sie müssen den Pfad eventuell an ihre Installation von Bug Tracker anpassen.

Das ist alles. Sie haben jetzt eine neue Sprache. Ich würde es begrüßen, wenn Sie mir die übersetzte String-Datei schicken; diese wird dann in neueren Versionen automatisch unterstützt.
