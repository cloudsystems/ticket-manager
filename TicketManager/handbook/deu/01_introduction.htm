1.1 Was ist Bug Tracker?

Bug Tracker ist ein (PHP) webbasiertes Fehlerverfolgungssystem. Es hilft ihnen ihre Softwareentwicklung zu managen. Es benutzt eine Datenbank, um Probleme (die gelöst werden müssen), Fehler (die behoben werden müssen) und Leistungsmerkmale (die hinzugefügt werden müssen), zu speichern.

Die meisten Menschen sind wie ich. Sie haben nur eine begrenzte Anzahl von "Speicherplätzen" für Fehler. Es ist einfach sich ein oder zwei Fehler zu merken. Aber wenn ein neuer Fehler hereinkommt wird ein älterer hinausgeworfen. Mit einem Fehlerverfolgungssystem können Fehler besser verwaltet werden. Die ganze Entwicklungshistorie einer Software kann in der Datenbank für spätere Einsichtnahme gespeichert werden. Eine Datenbank der Fehler ist für ein gutes Softwareentwicklungsteam unentbehrlich.

Bug Tracker hat zwei Benutzeroberflächen. Eine ist für das interne Entwicklungsteam, das andere für Kunden oder Endbenutzer, die hier ihr Feedback abgeben können. Er hat eine flexible Verwaltung für Benutzer-/Gruppenrechte, Such- und Filterfunktionen für die Datenbank, FAQ und Dokumentenverwaltung und mehr. Bitte sehen Sie sich den Rest dieses Dokuments an.

1.2 Leistungsmerkmale

  • SEHR SEHR SEHR einfach zu installieren
  • Einfach zu benutzen
  • Umfassendes Berechtigungssystem
  • Emailbenachrichtigung bei Aktualisierung eines Fehlerberichts
  • Oberfläche für Kundenfeedback
  • Oberfläche für Anonymes Feedback
  • Suche/Filter für jedes Feld
  • FAQ/Dokumentenmanagementsystem
  • Systemprotokoll, Benutzungsstatistik
  • Unterstützung für UTF-8
  • Mehrsprachige Benutzeroberfläche
  • Hochladen von Anhängen
  • HTML Editor im Textbereich
  • Unterstützung für PostgreSQL und MySQL
  • Unterstützung für das Senden von Emails über einen SMTP-Server der Authentifzierung erfordert

Bug Tracker ist ein leistungsstarkes Programm. Es ist gut getestet. Bei einem Test auf einem System mit einer AMD 1GHz CPU und 256 MB RAM unter FreeBSD 5.3, wurde festgestellt das das System mit 1000 Benutzern, 10000 Anonymen Benutzern, 10000 Fehlerberichtens, 10000 Dokumenten und 10000 FAQ's immer noch sehr flott läuft.

1.3 Die Geschichte von Bug Tracker

Bug Tracker wird von Wang, Chun-Pin (Alex Wang) entwickelt. Er hat ein Fehlerverfolgungs- und Kundenfeedback- System für Synology Inc entwickelt. Es war die Erste Version von Bugtracker. Das System das Synology benutzt hat, war relativ einfach, nicht sehr gut organisiert und hatte viele Synology - spezifische Funktionen. Das System läuft bereits seit mehr als 3 Jahren.

Nachdem Alex Synology zunächst verlassen hatte, gab er sich selbst einen langen Urlaub, um seine eigenen Projekte zu entwickeln. Er entwickelte einen FTP-Dienst und wendete sich der Open - Source zu. Er brauchte ein Fehlerverfolgungssystem. Das System, das er für Synology entwickelt hatte ist urheberrechtlich geschützt und hat viele Nachteile. Er wollte sowieso schon lange ein neues Fehlerverfolgungssystem schreiben. Also überarbeitete er den Entwurf und schrieb ein neues System - Bug Tracker. Bug Tracker ist gut organisiert. Es hat mehr Leistungsmerkmale, eine bessere Benutzeroberfläche und unterstützt mehrere Sprachen.

Das einzige Problem war: Bug Tracker hatte nur einen Benutzer. Also gab er es kostenlos an einige Firmen: Storlink Semiconductors, Inc., eine Tochtergesellschaft von ASUS. Und schließlich wurde es Open Source!
